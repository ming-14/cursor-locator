# 鼠标小圆圈 — 编码规范

## 1. Python 通用规范

### 1.1 缩进与格式
- 缩进：4 空格，严禁 Tab
- 行宽：不超过 100 字符
- 空行：模块级定义间 2 空行；类方法间 1 空行

### 1.2 导入顺序

```
# 1. Python 标准库
import sys
import os
import queue

# 2. 第三方库
import tkinter as tk
from tkinter import ttk
import ctypes
import ctypes.wintypes as w

# 3. 项目内部模块（统一绝对导入）
from src.core.config import Config
from src.infra.win32_api import *
```

禁止使用相对导入（`from .xxx import ...`），统一使用 `from src.层级.模块 import ...`。

### 1.3 命名约定

| 类型 | 风格 | 示例 |
|---|---|---|
| 模块/包 | snake_case | `ring_worker.py`、`settings_panel.py` |
| 类 | PascalCase | `MouseRingWindow`、`SettingsPanel` |
| 函数/方法 | snake_case | `_on_tick()`、`handle_message()` |
| 变量 | snake_case | `_hook_handle`、`_msg_queue` |
| 常量 | UPPER_CASE | `WS_EX_LAYERED`、`WM_HOTKEY` |
| 私有（模块/类内）| _prefix | `_tray_hwnd`、`_hook_proc_ref` |
| 特殊回调/外部函数 | _prefix | `_mouse_hook_proc`、`_ring_wnd_proc` |

## 2. 注释规范

### 2.1 模块头注释（Doxygen 风格）

每个 `.py` 文件顶部必须有 `@file` 和 `@brief`：

```python
"""
@file ring_worker.py
@brief 环线窗口线程：MouseRingWindow + 低层鼠标钩子 + 窗口过程
"""
```

### 2.2 函数/方法注释

复杂的函数需要注释，简单 getter/setter 可不加：

```python
def _on_tick(self, mouse_x, mouse_y):
    """每帧渲染逻辑：取色 → 碰撞检测 → 更新帧

    @param mouse_x 鼠标屏幕 X 坐标
    @param mouse_y 鼠标屏幕 Y 坐标
    """
```

### 2.3 代码注释
- 关键逻辑、边界条件、平台相关的 hack 处必须加注释
- 注释写在代码上方或行尾，与代码语言一致（中文）
- 不要写显而易见的注释（如 `x += 1  # x 加 1`）

```python
# 钩子模式：鼠标静止时不消耗 CPU，仅在移动时触发
if mode == 'hook':
    self._start_hook_tracking()
```

## 3. 项目特定规则

### 3.1 线程安全
- `Config` 使用 `Lock` 保护，非原子操作在单一线程串行
- `queue.Queue` 是唯一的跨线程数据通道
- Win32 窗口句柄仅在创建线程中操作（hwnd 线程亲缘性）
- Tkinter 控件仅在主线程操作

### 3.2 Win32 命名
- 常量/结构体/API 声明全部集中在 `src/infra/win32_api.py`
- Win32 常量保持原名称：`WS_EX_LAYERED`，不重命名
- API 包装函数加 `_` 前缀：`_SetWindowPos`、`_GetCursorPos`
- 避免 `from ctypes import *`，显式导入所需类型

### 3.3 导入限制
- `win32_api.py` 是纯声明文件，不应包含可执行逻辑
- `config.py`、`rendering.py` 零依赖原则（只依赖标准库）
- 禁止循环导入。如需，将共享逻辑提升到 `core/` 层

### 3.4 错误处理
- 文件/配置读取失败应静默处理，不中断程序
- Win32 API 调用失败必须检查返回值，失败时应有降级或日志
```python
_hook_handle = _SetWindowsHookExW(...)
if not _hook_handle:
    print('[警告] 无法安装鼠标钩子，回退到定时器模式')
```
